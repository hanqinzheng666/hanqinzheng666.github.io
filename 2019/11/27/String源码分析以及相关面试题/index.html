String源码分析

  • final修饰，不可被继承
  • 实现Serializable接口，可被序列化
  • 实现Comparable接口，可比较大小
  • final char[] value用于存储字符串数据(不可变性)

什么是Serializable接口

一个对象序列化的接口，一个类只有实现了Serializable接口，它的对象才能被序列化

什么是序列化？

序列化是将对象状态转换为可保持或传输的格式的过程。与序列化相对的是反序列化，它将流转换为对象。这两个过程结合起来，可以轻松地存储和传输数据。

为什么要序列化对象

  1. 把对象转换为字节序列的过程称为对象的序列化
  2. 把字节序列恢复为对象的过程称为对象的反序列化

什么情况下需要序列化?

  1. 当我们需要把对象的状态信息通过网络进行传输，或者需要将对象的状态信息持久化，以便将来使用时都需要把对象进行序列化
  2. Serializable只是个空接口,我们可以将它理解成一个标识接口。
  3. 在Java中的这个Serializable接口其实是给jvm看的，通知jvm，我不对这个类做序列化了，你(jvm)帮我序列化就好了。

结论（重要）

  • 常量与常量的拼接结果在常量池。且常量池中不会存在相同内容的常量。(常量池的值共享，堆中的值不共享)
  • 只要其中有一个是变量，结果就在堆中。（上图中的s4和s5）
  • 如果拼接的结果调用intern()方法，返回值就在常量池中
